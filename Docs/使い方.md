# Constraint Link Setup Tool — 簡単な使い方

## 概要
このツールは「義手（パーツ）」をアバターに追従させるための自動マッピングとコンストレイント適用を支援します。
通常は義手側のベースボーンだけを指定すれば自動でアバター側の対応ボーンを推定し、回転同期（Rotation Constraint）を設定します。必要に応じてアバター側ベースボーンを手動で上書き可能です。

## 必要条件
- Unity（バージョンはプロジェクト依存）
- nadena.dev.modular_avatar.core（`ModularAvatarBoneProxy` を使用）
- Unity の `UnityEngine.Animations` 名前空間（RotationConstraint 等）

## クイックスタート
1. Unity エディタメニューから `dennokoworks/Constraint Link Setup Tool` を開く。
2. Step 1 で「義手側ベースボーン（Prosthetic Base Bone）」を選択。
   - 多くの通常ケースではこれだけで OK。ツールは義手ベースボーンの親を自動的にアバター側ベースボーン候補として扱います。
3. 必要なら Step1 の「アバター側ベースボーンを手動指定する」をオンにして任意のアバターボーンを指定。
4. Step 2 で自動マッピングプレビューを確認。必要に応じて個々のペアの `使用` を切り替えられます。
5. Step 3 の `コンストレイント設定の実行` を押すと、Undo 対応でコンストレイントと `ModularAvatarBoneProxy` が追加されます。
6. 元に戻す場合は `コンストレイントの削除 (元に戻す)` ボタンを使用します（確認ダイアログあり）。

## 主要オプション（UI）
- `アバター側ベースボーンを手動指定する`：手動でアバター側ベースボーンを指定するかどうか。
- `左右の判定設定`：左右（L/R）を自動判定するか、強制するかを選択。
- `デバッグモード`：マッピング時の詳細ログを `Docs/Log/` に吐きます。

## 仕組み（簡潔）
- マッピングは以下の順序で行われます：
  1. ベースボーン同士をアンカーとしてペア化（義手のベース ↔ アバターの解決済みベース）
  2. アバター側をキャッシュ（HumanBodyBones 辞書 + 正規化名）
  3. 義手側を走査し、辞書ベース→正規化名ベースでマッチ
  4. フォールバックとして階層インデックスベースのマッチを試行
- サイド判定はボーン名の接尾辞/接頭辞（L/R/Left/Right 等）を優先し、識別できなければ位置情報に基づく推定を行います。

## 想定される義手（パーツ）の構造と運用ルール
- 推奨構成（最も簡単）:
  - 義手のベースボーン（例：義手の肩相当）を単体で選択すると、その親（Transform.parent）をアバター側のベース候補として自動解決します。
  - 義手が既にアバターの子オブジェクトとして配置されている場合も機能します（ツールは共通祖先を考慮して義手ルートを決定します）。

- 想定するボーン命名・構造:
  - ボーン名は一般的な命名規則（Left/Right, _L/_R, L_ など）に従っていると精度が高まります。
  - 数字付きや特殊プレフィックスが混在していても、内部で正規化してマッチングを行いますが、極端に破壊的な命名は一致を逸らす可能性があります。

- 自動解決の限界と対処:
  - 義手ベースボーンの親が存在しない（root 直下など）場合、自動解決は失敗します。その場合は UI の警告に従い手動指定してください。
  - 自動推定で期待どおりのボーンが選ばれない場合は「手動指定」を使って正しいアバターボーンを与えてください。

## 実装上の注意・制約
- 現状は Unity 標準の `RotationConstraint` を使用して回転同期を行います。
  - VRChat 固有の `VRC Rotation Constraint` と内部挙動が異なる場合があります。VRChat専用の挙動に合わせたい場合は別途切替実装が必要です。
- `ModularAvatarBoneProxy` を使用するため、対象プロジェクトに Modular Avatar のパッケージが含まれている必要があります。
- 回転オフセットは現在 Euler 角で扱っている箇所があるため、Gimbal 回転の影響を受ける可能性があります（必要に応じて Quaternion ベースの補正検討推奨）。

- Bone Proxy が想定と異なるターゲットに設定された：手動で `アバター側ベースボーン` を指定して再実行してください。
